Database Types

There are two types of databases that are used for web applications; Relational and Non-Relational databases. A relational database is a collection of data that is structured and accessed in order to locate specific information. It does this by maintaining a relationship between two or more entities. Some benefits of relational databases are its scalability, ease-of-use, integrity of the data and uptime/reliability of the database. Some downsides of relational databases are that it's slow to pull data, inability to store documents/graphs in original format, sparse data storage and fixed schema (unable to change the structure once created).

A non-relational database is a collection of data that is unstructured and doesn't have a strict distinction between rows, columns and their relations. Due to this increased flexibility, these databases are generally faster and more flexible. Some benefits of relational databases are its scalability, ease-of-use, speed, and processing time. Some downsides of relational databases are that it can struggle with large datasets, difficulty updating data, prone to unexpected behavior and lack of standardization between each of the vendors for non-relational databases.

How Do They Interface?

In order to connect your web application to a database, you first need to pick a DBMS, or Database Management System. From there, you can define your schema by laying out what type of data you'll need to hold on the back-end. After this initial work is completed, you can connect your database by using the following steps:

  • 1. Create the Database.
  • 2. Set up the server for your Database.
  • 3. Connect the Database to the Web Application.
  • 4. Create a Data Access Layer.
  • 5. Add measures to protect the data.
  • 6. Optimize the Database to work quickly.
  • 7. Test your Database works.
  • 8. Backup the Database regularly to minimize data losses.
